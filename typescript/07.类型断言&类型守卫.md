# 类型断言&类型守卫
> 有些情况下TS并不能准确得推断类型，这个时候可能产生不必要的警告  
> 但是不要滥用类型断言，因为强制的类型断言会使TS丧失代码提示的能力  

```ts
const person = {};

person.name = 'xiaomuzhu'; // Error: 'name' 属性不存在于 ‘{}’
person.age = 20; // Error: 'age' 属性不存在于 ‘{}’

// 使用类型断言
interface Person {
    name: string;
    age: number;
}

const person = {} as Person;
person.name = 'xiaomuzhu';
person.age = 20;
```
